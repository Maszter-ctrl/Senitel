1. System Preparation

Before building Sentinel, ensure your Linux system is up to date:

sudo apt update
sudo apt upgrade

Install essential build tools:

sudo apt install build-essential git curl wget unzip

This ensures your system has compilers, make, and unzip utilities required for building and running Python binaries.

2. Extracting the Source ZIP

Download the source ZIP from the Sentinel main page, then extract it:

unzip blocker_source.zip -d sentinel_source
cd sentinel_source

Make sure the folder path contains **no spaces or special characters** to avoid issues with PyInstaller.

3. Python Installation

Sentinel requires Python 3.11 or higher. Install it on Linux:

sudo apt install python3.11 python3.11-venv python3.11-dev

Verify the version:

python3.11 --version

If your system doesn’t have 3.11, consider using deadsnakes PPA or compiling from source.

4. Virtual Environment Setup

python3.11 -m venv venv
source venv/bin/activate

Using a virtual environment prevents dependency conflicts with system Python packages.

5. Installing Dependencies

pip install --upgrade pip
pip install -r requirements.txt

All required Python libraries for Sentinel will be installed in your virtual environment.

6. Customizing the Blocklist

Edit blocklist.json to add processes you want Sentinel to block:

{
  "processes": ["example_process1", "example_process2"]
}

You can also modify the CLI interface to dynamically add/remove processes.

7. Building the Linux Binary

Use PyInstaller to create a single executable:

pyinstaller --onefile blocker.py --name blocker_linux

Output executable will appear in the dist folder.

Test the binary in a safe environment first.

8. Optional: Build for Windows/macOS

pyinstaller --onefile blocker.py --name blocker_windows.exe
pyinstaller --onefile blocker.py --name blocker_mac

Cross-building may require virtual machines or containers for macOS/Windows.

9. Testing Your Build

  • Run in a safe VM or container.
  • Check logs for blocked processes.
  • Ensure no critical system processes are blocked.

10. Advanced Configurations & Tips

  • Use cron jobs to restart Sentinel on reboot.
  • Integrate with systemd for daemon management.
  • Configure logging levels for different environments.
  • Automate blocklist updates via external JSON URLs.
  • Advanced CLI flags for debug, dry-run, or verbose monitoring.
  • Custom scripts to dynamically add/remove processes while running.

11. FAQs and Troubleshooting

Q: Why is PyInstaller failing on my system?
A: Make sure Python 3.11 and development headers are installed, and you are using a clean virtual environment.

Q: Sentinel crashes immediately?
A: Check your blocklist — avoid system-critical processes.

Q: How do I update Sentinel?
A: Rebuild from latest source ZIP or pull latest repository changes.

12. Safety & Best Practices

  • Always backup critical data.
  • Run initial tests in a virtual machine or container.
  • Do not run as root unless absolutely necessary.
  • Inspect source code if running binaries from untrusted sources.
  • Keep your system updated to avoid unexpected issues.

Sentinel is powerful. Misconfiguration can terminate essential processes. Always test carefully.
